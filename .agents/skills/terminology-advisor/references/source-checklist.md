# Source checklist

Use official sources and project artifacts. Search or open the latest source before answering version-specific questions.

## Canada
- Canada Health Infoway InfoCentral standards pages: SNOMED CT Canadian Edition, pCLOCD, terminology gateway/server, release notes.
- Canadian FHIR Registry: national, jurisdictional, and local FHIR profiles, extensions, value sets, URIs, and implementation guides.
- CIHI codes, classifications, and terminologies: ICD-10-CA, CCI, Canadian Coding Standards, Canadian Classifications Browser, change tables.
- Provincial/territorial sources: Ontario Health/eHealth Ontario, Alberta Health/AHS, BC Ministry of Health/PHSA, MSSS/RAMQ, and relevant provincial program sites.

## United States
- HL7 published US Core Implementation Guide; avoid treating continuous-build pages as final unless asked.
- NLM VSAC and VSAC FHIR terminology service for value sets and supported code systems.
- ASTP/ONC USCDI and Interoperability Standards Platform vocabulary guidance.
- NLM RxNorm and UMLS resources.
- CDC PHIN VADS, CVX/MVX, race/ethnicity code sets, and public-health IG terminology.
- CMS ICD-10-CM/PCS and HCPCS; AMA CPT licensing/source material where applicable.

## International/steward sources
- HL7 FHIR terminology module and HL7 Terminology (THO).
- SNOMED International documentation, release notes, URI standard, implementation guides, and SNOMED CT terminology service guidance.
- LOINC/Regenstrief release pages, FHIR terminology service documentation, answer lists, groups, and panels.
- UCUM specification and unit validation resources.
- WHO ICD resources where the context is international rather than Canadian or U.S. realm.

## Source evaluation questions
- Is this the current published artifact or a continuous/ballot build?
- Does the artifact apply to the user's country, province/state, and program?
- Does the artifact specify FHIR version and package version?
- Are terminology versions pinned or floating?
- Are expansions point-in-time snapshots?
- Are there licensing restrictions on display, reproduction, or download?
- Is the terminology server authorized to serve the needed edition/version?
